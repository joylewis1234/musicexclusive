

## Create Backend Audit Documentation

### Overview
Create a `docs/backend-audit/` folder with the first Markdown file containing the Security Risk Map document that tracks the current security hardening status.

### Files to Create

**`docs/backend-audit/security-risk-map.md`**
- Contains the full Security Risk Map organized by severity: Critical, High, Medium, Low/Deferred
- Documents each identified risk, its fix status, mitigation applied, and any residual concerns

### No Other Changes
- No database changes
- No edge function changes
- No frontend changes
- This is purely a documentation addition
